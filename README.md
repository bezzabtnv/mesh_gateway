# Meshtastic Telegram Gateway

[![Python 3.7+](https://img.shields.io/badge/python-3.7+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Шлюз для интеграции сети **Meshtastic** с мессенджером **Telegram**. Позволяет отправлять и получать сообщения из LoRa‑сети через Telegram‑бота, отслеживать статус доставки, автоматически отвечать на ping‑запросы и мониторить состояние батареи устройства.

> [!NOTE]
> Для работы необходим **узел Meshtastic с включённым TCP‑сервером** (порт 4403) и **Telegram‑бот**, созданный через [@BotFather](https://t.me/botfather).

---

## 📌 Возможности

- **Два режима работы**  
  - 📡 **Радио** – отображение всех сообщений из эфира (каналы 0 и 1) в Telegram.  
  - 🎯 **Мост** – двусторонняя связь с конкретной нодой Meshtastic (отправка/получение сообщений, подтверждения доставки).
- **Поддержка ping** – автоматический ответ на ping‑запросы с указанием количества хопов.
- **Статусы доставки** – реакции Telegram (🕊️ → 👍 → 🤷) и отображение времени доставки.
- **Мониторинг батареи** – уведомления о низком и критическом уровне заряда.
- **История сообщений** – просмотр последних сообщений из эфира.
- **Многопользовательский режим** – поддержка нескольких чатов (с возможностью ограничения по списку ID).
- **Автоматическое переподключение** – восстановление связи с Meshtastic при обрыве.
- **Конфигурация через файл** – все настройки хранятся в `config.json`.

---

## 📋 Требования

- Python 3.7 или выше
- Устройство Meshtastic с включённым TCP‑сервером (порт 4403)
- Telegram‑бот, созданный через [@BotFather](https://t.me/botfather)
- Доступ к сети (для связи с Meshtastic и Telegram API)

## 🔧 Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yourusername/meshtastic-telegram-gateway.git
   cd meshtastic-telegram-gateway
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
   Если файла `requirements.txt` нет, установите вручную:
   ```bash
   pip install telebot meshtastic pubsub requests
   ```

3. Запустите скрипт для создания конфигурации:
   ```bash
   python mesh_gateway.py
   ```
   Скрипт предложит ввести данные: токен бота, IP‑адрес Meshtastic, разрешённые чаты, целевую ноду и другие параметры. Конфигурация сохранится в `config.json`.

---

## ⚙️ Конфигурация

Файл `config.json` имеет следующую структуру:

```json
{
  "telegram_token": "ваш_токен",
  "meshtastic_host": "192.168.1.100",
  "allowed_chat_ids": ["123456789", "987654321"],
  "target_node": "!12345678",
  "ping_channel": 1,
  "max_message_size": 200,
  "max_hops": 7,
  "restart_on_crash": true,
  "battery_low_threshold": 3.5,
  "battery_critical_threshold": 3.3,
  "battery_check_interval": 300
}
```

| Параметр | Описание |
|----------|----------|
| `telegram_token` | Токен вашего Telegram бота. |
| `meshtastic_host` | IP‑адрес устройства Meshtastic (порт 4403). |
| `allowed_chat_ids` | Список ID чатов, которым разрешено использовать бота. Если пустой – доступ открыт всем. |
| `target_node` | ID целевой ноды для режима моста (формат `!xxxxxxxx`). Если не указан, режим моста недоступен. |
| `ping_channel` | Номер канала (0–7), на котором бот слушает и отвечает на ping. |
| `max_message_size` | Максимальный размер сообщения в байтах (обрезается при превышении). |
| `max_hops` | Максимальное количество хопов, отображаемое в истории. |
| `restart_on_crash` | Автоматический перезапуск при критической ошибке. |
| `battery_low_threshold` | Порог низкого заряда батареи (вольты). |
| `battery_critical_threshold` | Порог критического заряда батареи (вольты). |
| `battery_check_interval` | Интервал проверки батареи (секунды). |

## 🚀 Запуск

После настройки запустите шлюз:

```bash
python mesh_gateway.py
```

При успешном запуске в консоли появится информация о подключении и текущих настройках. Шлюз автоматически подключается к Meshtastic и Telegram, после чего готов к работе.

## 🤖 Режимы работы

### 📡 Режим радио

- Отображает все сообщения из каналов 0 и 1.
- Сообщения приходят в формате:
  ```
  📡 CH0 [!12345678] 14:35:12
  Привет всем!
  ```
- Поддерживает автоматический ответ на ping (если включён).

### 🎯 Режим моста

- Требует указания `target_node` в конфигурации.
- Обеспечивает двустороннюю связь с выбранной нодой.
- При отправке сообщения в Telegram бот ставит реакцию 🕊️.
- При получении подтверждения (ACK) от ноды реакция меняется на 👍 и приходит время доставки.
- Если подтверждение не получено в течение 5 минут, реакция становится 🤷.

## ⌨️ Команды и кнопки

Бот поддерживает как текстовые команды, так и интерактивную клавиатуру.

### Основные команды

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие и показ меню |
| `/menu` | Показать клавиатуру с кнопками |
| `/radio` | Переключиться в режим радио |
| `/gateway` | Переключиться в режим моста (доступен только если указана `target_node`) |
| `/status` | Статус подключения, заряд батареи, режим, счётчик ping |
| `/history` | Последние 10 сообщений из эфира |
| `/help` | Справка |
| `/restart` | Перезагрузка шлюза |
| `/pingon` | Включить автоматический ответ на ping |
| `/pingoff` | Отключить ответ на ping |
| `/pingchannel` | Установить канал для ping (0–7) |

### Кнопки меню

| Кнопка | Действие |
|--------|----------|
| 📡 Режим радио | Переключение в режим радио |
| 🎯 Режим моста | Переключение в режим моста (только если есть `target_node`) |
| 📊 Статус | Отображение текущего состояния |
| 📜 История радио | Показ последних сообщений из эфира |
| ℹ️ Помощь | Краткая справка |
| 🔄 Перезагрузить | Перезапуск шлюза |
| 🚫 Ping: Выкл / ✅ Ping: Вкл | Включение/отключение автоответа на ping |
| 📻 Ping канал: N | Циклическое переключение канала для ping |

## 🔋 Мониторинг батареи

Шлюз периодически опрашивает устройство Meshtastic и получает данные о напряжении батареи. При снижении напряжения ниже порогов (`battery_low_threshold` и `battery_critical_threshold`) в разрешённые чаты отправляются уведомления:

- 🟡 **НИЗКИЙ УРОВЕНЬ БАТАРЕИ: 3.45V**
- 🔴 **КРИТИЧЕСКИЙ УРОВЕНЬ БАТАРЕИ: 3.28V**

Интервал проверки и пороги настраиваются в `config.json`.

---

## 📝 Логирование

Все ошибки записываются в файл `gateway.log` (уровень `ERROR`). В консоль выводятся только критические сообщения. Для отладки можно изменить уровень логирования в коде на `INFO` или `DEBUG`.

> [!TIP]
> Чтобы временно включить подробный вывод, замените в начале файла `mesh_gateway.py` строки `app_logger.setLevel(logging.ERROR)` на `app_logger.setLevel(logging.DEBUG)`.
